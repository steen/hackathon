# Generate Plans Index

Create or overwrite `specs/plans/README.md` as an index of all generated plans.

## Steps

1. List all top-level phase files matching `specs/plans/phase-*.md` (exclude files inside subdirectories).
2. For each phase file, read it and extract the phase number, title, and `Status:` value.
3. List feature files inside each matching `specs/plans/phase-{N}/` directory.
4. Use the Write tool to write `specs/plans/README.md` with the content template below.

## File template

```markdown
# Implementation Plans

Generated by the `prd-to-plans` workflow. Re-run to regenerate after PRD changes.

| Phase | Title | Status | Plan |
|-------|-------|--------|------|
| 1 | ... | planned | [phase-1-...](phase-1-....md) |
| 2 | ... | planned | [phase-2-...](phase-2-....md) |

---

## Phase 1: {title}
Status: planned · [Phase plan](phase-1-{slug}.md)

Features:
- [{feature name}](phase-1/feature-{slug}.md)

## Phase 2: {title}
...
```

When done, print `wrote: specs/plans/README.md`.
