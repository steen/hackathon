### Fixed

- `apps/web/src/routes/Chat.tsx`: close the empty-channel-hint render race. `useMessages`'s `setHistoryLoading(true)` lands in a `useEffect` that fires post-commit, so the render where `activeChannel` first flips to a real id has `historyLoading=false` and `messages=[]` — all four `showEmptyChannelHint` conditions briefly hold and the "start of channel" hint flashes for one frame on slower runners (CI). The gate now also requires the hook to have acknowledged the active channel (signalled by `historyLoading=true` or `messages.length > 0`), tracked via the React "adjusting state during rendering" pattern. The matching test in `Chat.test.tsx` swaps its single-frame snapshot for a `MutationObserver` across the whole in-flight window so the regression caught by CI on PR #992 stays caught.
