### Added

- Web "Direct messages" sidebar section listing the viewer's DM
  conversations (Phase 9 #874). Rows sort by `last_message_at DESC`,
  carry per-row + per-section unread badges, and the section's "+ New
  DM" button opens a Phase-8-Modal-based user picker that POSTs
  `/api/dms` for find-or-create.
- `useDMs` hook owns the DM listing, dm-frame upserts (decision §8
  self-sufficient first contact), local unread increments
  (decision §12), and reconcile-overwrite on listing fetch / WS open.
  Cross-tab `{type:"read", scope:"dm"}` frames zero local badges
  without a refetch (decision §7).
- `DMThread` renders the active DM history + composer, sends via
  `POST /api/dms/{id}/messages`, and wires
  `useReadMarker("dm", conversationId)` so the latest peer message id
  advances the server-side read pointer (debounced, focus-flushed).
