## Phase 10 — keys-RPC + lazy-wrap-on-online (#984)

- `POST /api/channels/{id}/keys` ships the three modes from
  decision-log §8: bootstrap (first user wraps the channel root key
  to themselves), fill-in (existing member supplies a missing wrap
  for a current-generation recipient), rotation (caller covers every
  current member at `MaxGen + 1`). Server selects mode from
  `generation_id` vs. `max(channel_keys.generation_id)`; mismatch
  returns 400 `invalid_generation`. L30 + L39 byte-length checks fire
  on every wrap-insert path.
- `GET /api/channels/{id}/members/wraps-needed` returns the L22
  shape: per-row `MembershipBlock` with pinned pubkeys + signature,
  plus convenience username / box / sign pubkey fields from the live
  users row. `is_public` is server-resolved per L38 so the verifier
  doesn't depend on a stale channel-listing cache. Server-side
  rate-limit `wraps-needed-read` (burst 10 / refill 1 minute, L31 +
  L36) gates the endpoint per user.
- WS frames `members_changed` (rotation, fanned `BroadcastAll`) and
  `key_received` (fill-in / rotation, fanned to the recipient's
  `user:<viewer>` topic only) are emitted from the keys-RPC handler
  per L9. Wire types were declared in #978; this PR populates the
  emitter side.
- Web hook `useChannelKeys` runs the L14 lazy-wrap-on-online query on
  every WS open (L31 60s debounce + once-per-channel cache); clears
  per-channel `waitingForKey` state on a `key_received` frame. The
  compute-and-POST arm is intentionally stubbed — the channel root
  key plaintext is only available after the encrypted-message
  decrypt path lands with #983.
- CLI `chatd watch` calls `wraps-needed` on connect and surfaces the
  missing-wrap count on stderr (same L31 debounce shared across
  reconnects). The compute-and-POST arm follows #983.
- Mirrored types in `packages/api-client/src/types.ts` +
  `packages/go-client/channels.go`: `WrapsNeededRow`,
  `WrapsNeededResponse`, `PostChannelKeysBody`,
  `PostChannelKeysResponse`. New `Client.WrapsNeeded` +
  `Client.PostChannelKeys` (Go) and `HttpClient.wrapsNeeded` +
  `HttpClient.postChannelKeys` (TS) methods.
- e2e coverage under `tests/e2e/phase-10/keys-rpc/`: bootstrap of
  `#general`, fill-in delivery + race-loss 409, L39
  `wrap_size_invalid`, `invalid_generation`, non-member 403 on both
  endpoints.
