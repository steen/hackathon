- DMs: ship CRUD surface — `POST /api/dms` (idempotent find-or-create,
  201/200 per L18), `GET /api/dms` (hides empty conversations,
  `last_message_at DESC`, viewer-relative `peer` + `unread_count`),
  `POST /api/dms/{id}/messages` (4096-byte cap, `dm-write` per-user
  bucket, atomic sender `dm_reads` advance via `InsertDMMessageTx`),
  and `GET /api/dms/{id}/messages` (ULID-cursor newest-first paging).
  Each message broadcasts a self-sufficient `{type:"dm"}` frame to both
  participants' `user:<viewer>` topics so a recipient's first DM
  renders without a `GET /api/dms` round-trip.
