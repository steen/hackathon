### Fixed

- WebKit (iPhone-class) phone-width playwright failure (#634). The mobile single-column layout stacked an unbounded sidebar above the messages region; when the sidebar's content (channel list, presence list) was tall enough — easily reproduced inside a Playwright run as test users accumulate channels in the same DB — it pushed the messages region completely off-screen, and Playwright reported `.messages` as "hidden". Cap the mobile sidebar at `max-height: 40vh` and let it scroll inside; messages takes the remaining space and stays visible regardless of channel-list length. Also rewrites the mobile body from `display: grid` to `display: flex; flex-direction: column` since the nested grid-inside-flex sizing path was where webkit and chromium diverged in the first place.
