### Added

- `.env.example` at the repo root — copy-pasteable starter template covering every server env var read at startup (`CHAT_JWT_SECRET`, `CHAT_INVITE_CODE`, `CHAT_DB_PATH`, `CHAT_LISTEN_ADDR`, `CHAT_ALLOW_PUBLIC_BIND`, `CHAT_ALLOWED_ORIGINS`, `CHAT_TRUSTED_PROXY`, `CHAT_LOG_LEVEL`, `CHAT_SERVER_PORT`, plus a `CHAT_BCRYPT_COST` annotation deferred to #785). Each entry documents the failure mode if missing/invalid so a fresh operator hits the validator with sensible inputs on first boot.
- `scripts/check-env-example.mjs` — hand-runnable drift check that mirrors `scripts/check-workspace-exports.mjs`. Scans `Env*` consts in `apps/server/internal/config/config.go` plus the legacy `*Env` const block in `apps/server/main.go` and exits non-zero with a named offender list when `.env.example` is missing a CHAT\_\* variable.
- README "Quick start" now references `cp .env.example .env`; the "Server environment variables" table grew rows for `CHAT_TRUSTED_PROXY`, `CHAT_LOG_LEVEL`, and `CHAT_BCRYPT_COST`, all of which were already read by the server but undocumented.

### Changed

- `.gitignore` ignores the root-anchored `/.env` only (not `*.env`), so the tracked `.env.example` template is unaffected.
