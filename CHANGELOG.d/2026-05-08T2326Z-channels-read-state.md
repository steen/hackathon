### Added

- `apps/server`: Phase 9 per-viewer channel read state. `GET /api/channels` now materializes a `channel_reads` row per non-empty channel for the authenticated viewer (decision log §11) and returns `last_message_id`, `last_message_at`, `last_read_message_id`, and `unread_count` per channel. Listing order is `last_message_at DESC NULLS LAST` (decision log §9 — activity-ordered). New endpoint `POST /api/channels/{id}/read` advances `channel_reads.last_read_message_id` (advance-only per L5), broadcasts a `{type:"read", scope:"channel"}` frame to the caller's `user:<viewer>` topic for cross-device sync (L10), and returns 204. The per-user `read-mark` token bucket (`ratelimit.ReadMarkUserConfig`, Burst=50/Refill=1m, L17) gates the endpoint. New e2e suite under `tests/e2e/phase-9/channels-read-state/` covers listing additive fields + ordering, materialize-on-first-list, post-baseline unread bumps, the WS read frame round-trip, the unknown-channel 404 branch, and the 51st-request 429.
