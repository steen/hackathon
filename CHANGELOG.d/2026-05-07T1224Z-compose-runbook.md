- **ops:** added `docker-compose.yml` at repo root and `docs/ops/runbook.md`. The compose file is single-service (`chat-server`), uses `build: .`, mounts a named `chat-data` volume at `/data`, and intentionally ships no `healthcheck:` block (distroless image has no shell; tracked by #796). The runbook covers prerequisites, first run, first-user registration, daily ops, restart-loop recovery, backup/restore (with the WAL caveat and the distroless `sqlite3` gotcha), reverse-proxy topology and the single-instance constraint, the `docker inspect` env-var caveat, upgrade, and the `build: .` rebuild trade-off. README points at the runbook for the full deploy story. Closes #792.
