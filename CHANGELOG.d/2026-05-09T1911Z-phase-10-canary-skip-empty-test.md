### Tests

- `apps/web/src/auth/AuthContext.test.tsx`: lock the wrong-passphrase canary's empty-`sign_pubkey` skip path (`AuthContext.tsx:90-105`). Two new cases drive `login(username, password, identityPassphrase)` against a server response whose `sign_pubkey` is absent or explicitly `""` and assert the call resolves without `WrongPassphraseError`, the JWT is persisted, `writeIdentitySeed` is still invoked so a later /me wave can register the keys, and `logout()` is NOT called. Without this, a future PR that swaps the comparison or drops the `remoteSignPub !== ""` guard would silently lock out every legacy user (NULL `sign_pubkey` from migration `0006_encryption.sql`) without breaking any test. `deriveIdentity`/`b64`/`ready` and `writeIdentitySeed`/`clearIdentitySeed` are mocked at the module boundary so the test pays no Argon2id cost and does not require libsodium-wrappers-sumo init under jsdom.
