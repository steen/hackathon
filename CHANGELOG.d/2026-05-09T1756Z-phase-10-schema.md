### Added

- `migrations/0006_encryption.sql`: phase-10 schema for end-to-end encryption. Adds the `channel_members`, `channel_keys`, and `dm_conversation_keys` tables; envelope columns (`cipher_suite`, `key_generation_id`, `nonce`, `ciphertext`, `sender_sign_pubkey`, `signature`, `client_created_at`) on `messages` and `dm_messages`; `box_pubkey` and `sign_pubkey` on `users`; `is_public` on `channels`; the `idx_channel_members_null_sig` partial forensic index (decision-log L33); and a case-insensitive unique index on `users.username` (decision-log L37). Per L23, the L21 `body`-to-envelope cutover ships in #983 — `body` and the new envelope columns coexist in this PR so wave-2/3/4/5 sub-issues build green.
- `apps/server/boot.go`: L18 boot guard between `appdb.Open()` and `appdb.Apply()`. Refuses startup against a pre-encryption `chat.db` that still holds plaintext rows; the operator-facing message lists all three SQLite WAL files (`chat.db`, `chat.db-wal`, `chat.db-shm`).
- `apps/server/internal/repo/channel_members.go`: skeleton with `InsertChannelMember` enforcing the L33 NULL-signature rule (private channels reject NULL `inviter_signature`). Read-side helpers return a sentinel `not implemented` error until #981 lands.
- `apps/server/internal/repo/channel_keys.go` and `apps/server/internal/repo/dm_conversation_keys.go`: type-only skeletons returning `not implemented`. #982 / #984 / #985 fill in the read and write paths.

### Changed

- `apps/server/internal/repo/channels.go`: `CreateChannel` gains an `isPublic bool` parameter (decision-log L24). `apps/server/internal/seed/seed.go` passes `true` for `#general` so new-user auto-add at registration time targets it; `apps/server/internal/http/channels_handlers.go` passes `false` until the membership creator surface exposes the toggle (#981).
