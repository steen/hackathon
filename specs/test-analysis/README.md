The agent processes one feature per tick. PRs are opened against `main` from `test-analysis/<date>-<sha>` branches. The orchestrator filters out features that already have an open agent PR so multiple ticks at the same `origin/main` SHA produce one PR per feature.

## How to run the agent

In a Claude Code session at the repo root:

```
/loop 90s /test-watch
```

The first tick after the loop starts will create `.claude/worktrees/test-agent` and `.claude/test-agent/state.json`, then check whether there's new work. Most ticks are silent no-ops. When the agent has work to do, it opens a PR with new E2E tests for one feature plus the regenerated findings docs.

## How to run the analysis once, manually

```
/test-analyze
```

writes findings to this directory without creating a branch or opening a PR. Useful for ad-hoc inspection. Pair with `/test-implement` to write the tests, then a normal `git commit` + `gh pr create` if you want to ship the analysis yourself.
